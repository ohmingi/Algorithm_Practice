<h2 data-ke-size="size26">백준 10799번 쇠막대기 (C++) : <a href="https://www.acmicpc.net/problem/10799" target="_blank" rel="noopener&nbsp;noreferrer">https://www.acmicpc.net/problem/10799</a></h2>
<h3 data-ke-size="size23">🧩 문제 요약</h3>
<ul style="list-style-type: disc;" data-ke-list-type="disc">
<li>괄호로 된 입력 문자열에서 <code>()</code>는 레이저</li>
<li>그 외 <code>(</code>는 쇠막대기 시작, <code>)</code>는 쇠막대기 끝</li>
<li>레이저는 현재 열려있는 쇠막대기들을 자른다</li>
<li>최종적으로 잘려진 <b>조각의 총 개수</b>를 구해야 한다</li>
</ul>
<h3 data-ke-size="size23">💻 코드</h3>
<pre class="arduino"><code>#include &lt;iostream&gt;
#include &lt;stack&gt;
#include &lt;string&gt;

using namespace std;

int main(){
stack&lt;char&gt; s;
string str;
cin &gt;&gt; str;

    int result = 0;
    bool b = true;

    for(int i = 0 ; i &lt; str.size() ; i++){
        if(str[i] == '('){
            s.push(str[i]);
            b = true;
        }
        else{
            if(s.top() == '('){
                s.pop();
                if(b == false){
                    result += 1;
                }
                else{
                    result += s.size();
                }
            }
            b = false;
        }
    }

    cout &lt;&lt; result;
}
</code></pre>
<h3 data-ke-size="size23">🔍 로직 설명</h3>
<ul style="list-style-type: disc;" data-ke-list-type="disc">
<li><code>(</code>가 나오면 스택에 push &rarr; 쇠막대기 시작 or 레이저 준비</li>
<li><code>)</code>가 나왔을 때 직전 문자가 <code>(</code>이면 &rarr; 레이저<br />&rarr; 스택 pop 후, 현재 열린 쇠막대기 개수만큼 조각 추가&nbsp;<br />즉, '(' 개수만큼 추가</li>
<li><code>)</code>가 나왔을 때 직전 문자가 <code>)</code>이면 &rarr; 쇠막대기 끝<br />스택에 남아있는 '('를 제거해서 막대기 하나가 끝났다는것을 설명<br />&rarr; 스택 pop 후 조각 +1, (막대기를 한번 나누면 1개가 아닌 2개가 되니 레이저수만큼 나누고 마지막에 1을 더해줘야 <br />그 막대기가 몇개로 나뉘어졌는지 정확히 알 수 있음)</li>
</ul>
<p data-ke-size="size16"><code>bool b</code>는 <b>직전 문자가 무엇인지</b>를 기억하기 위한 플래그다.</p>
<h3 data-ke-size="size23">⚠️ 주의한 점</h3>
<ul style="list-style-type: disc;" data-ke-list-type="disc">
<li>레이저와 쇠막대기 끝을 구분하는 것이 핵심</li>
<li><code>s.top()</code>을 검사하기 전에 스택이 비어있을 가능성은 없었지만,<br />안전하게 할 거면 <code>if (!s.empty())</code> 체크도 고려할 수 있음</li>
<li>괄호 문자열 처리에서 실수가 잦은 유형</li>
</ul>
